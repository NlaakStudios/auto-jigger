Modular Design

Everything about Maestro is modular. The goal is modular with maximum amount of Re-usability and performance. Whether it be the PHP script that creates the HTML5/CSS3 document that is the basis for all CloudApps, or an Applet, a Component or entire Library. From Graphics and Sounds, to Fonts and Glyphs...It is all designed to be Modular, Efficient and Fast.

Prototypal Inheritance

Maestro also provides the Interface and Inheritance foundation that is used by the Nano Framework as well as the Developer API and 2DGame Engine API. Interfaces are similar to Microsoft COM Objects (Think DirectX). Every CloudApp, Applet and Component has a UUID that represents that publishers software and allows for easy upgrading but also allows your to request a older version if a comparability problem arises.

Applets &  Components

Applets, Hmm... Ever watch a good action Movie notice the screen mess up and you see blocks? MPEG updates just what has changed, That's kind of what applets are. They allow you to break your Application or document up into smaller reusable parts and also allows you to embed the styling and JavaScript directly - an to top it off they are compiled, encoded and compressed then are ready for use.
